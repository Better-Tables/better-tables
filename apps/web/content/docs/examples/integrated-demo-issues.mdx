This document outlines known issues and a testing checklist for the integrated virtualized table demo.

## Current Issues

The `integrated-virtualized-table-demo.tsx` file has several API mismatches that need to be resolved before it can be properly tested. The main issues are:

### 1. FilterState API Mismatch
- **Issue**: Code uses `filter.value` but `FilterState` type uses `values` array
- **Error**: `Property 'value' does not exist on type 'FilterState'. Did you mean 'values'?`
- **Location**: Lines 193, 195, 197, 199 in filtering logic

### 2. PaginationManager API Mismatch
- **Issue**: Code calls `updateTotalItems()` method that doesn't exist
- **Error**: `Property 'updateTotalItems' does not exist on type 'PaginationManager'`
- **Expected**: Should use `setTotal()` method instead

### 3. Column Property Names
- **Issue**: Code references `column.label` but should use `column.displayName`
- **Error**: `Property 'label' does not exist on type 'ColumnDefinition'`
- **Location**: Line 387 in sorting display

### 4. UI Component API Mismatches
- **FilterBar**: Expects different props than provided
- **ActiveFilters**: Different prop interface than expected
- **Issue**: Components may not exist or have different APIs than assumed

### 5. HTML Entity Issue
- **Issue**: Unescaped `>` symbol in JSX
- **Error**: `Unexpected token. Did you mean '{'>'}' or '&gt;'?`
- **Location**: Line 450
